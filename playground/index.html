Get Started in 30 Seconds

Choose your mode. Each one is a single file.

Mode 1: Patch Your Framework

Already using Bootstrap? Add one line. Nothing else changes.

<!-- After bootstrap.min.css -->
<link rel="stylesheet"
  href="https://playground.cbrowser.ai/cdn/fix-bootstrap.css">

Also available: fix-tailwind.css, fix-shadcn.css, fix-material.css

Mode 2: Use as Your Framework

Replace Bootstrap/Tailwind entirely. Beautiful + accessible by default.

<link rel="stylesheet"
  href="https://playground.cbrowser.ai/cdn/cbrowser-ui.css">
<script type="module"
  src="https://playground.cbrowser.ai/cdn/cbrowser-ui.mjs">
</script>

10kb CSS + 5kb JS. Zero dependencies.

Mode 3: npm Install

For build systems. Import what you need.

npm install cbrowser-ui
// ESM
import 'cbrowser-ui/css';
import { FocusTrap } from 'cbrowser-ui';

// Or just the patch
import 'cbrowser-ui/fix-bootstrap';

Framework Patches

Each patch targets the specific CIF-failing components in that framework. Before → After.

What fix-bootstrap.css changes

Before

  • .btn-sm is 32px tall (fails touch target)
  • .nav-link is 36px (fails touch target)
  • .carousel ignores prefers-reduced-motion
  • .text-muted is 4.6:1 contrast (fails AAA)
  • Links in content not underlined (fails 1.4.1)
CIF: 52

After (one file added)

  • All buttons 44px minimum
  • Nav links 44px with proper spacing
  • Carousel respects motion preferences
  • Muted text at 7:1 contrast
  • Content links underlined
CIF: 94

Themes

Override 20 CSS variables. The entire design changes. Accessibility stays perfect.

:root {
  --cui-primary: #7c3aed;        /* Purple instead of blue */
  --cui-primary-fg: #ffffff;
  --cui-primary-hover: #6d28d9;
  --cui-radius: 1rem;            /* Rounder corners */
  --cui-font: 'Inter', sans-serif;
}

Every color combination is validated against WCAG AAA (7:1) for normal text and AA (4.5:1) for large text. The theme editor (coming soon) prevents you from picking colors that break contrast.
